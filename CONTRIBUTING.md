# Contributing to NEXIS

Contributions are welcome.

## Before you contribute

- Read the project documentation in `README.md`, `PRD.md`, and `GLOSSARY.md`.
- Keep changes focused and avoid unrelated churn.
- Follow the repository instructions and existing code style.

## Developer Certificate of Origin

By contributing to this project, you certify that you have the right to submit your contribution under the repository's public source-available license terms and that you agree to the Developer Certificate of Origin, Version 1.1.

Use a signoff line in each commit message:

```text
Signed-off-by: Your Name <you@example.com>
```

If you use Git from the command line, this is typically:

```bash
git commit -s
```

## Contribution licensing

Unless explicitly agreed otherwise in writing, contributions are accepted under the same repository licensing model:

- PolyForm Noncommercial License 1.0.0 for the public source-available distribution
- separate commercial licensing available from the project owner

## Pull requests

- Describe the problem being solved and the chosen approach.
- Update relevant documentation when behavior, workflow, or terminology changes.
- Add or update tests when code behavior changes.

Additional contribution guidelines can be expanded here as the project grows.
